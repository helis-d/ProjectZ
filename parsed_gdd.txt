Project Z – Game Design Document (GDD)
# Section 1: Core Gameplay Mechanics
## Game Loop and Round Structure
The game is a tactical FPS between Attack and Defense teams. Teams compete to be the first to win a set number of rounds.
Pistol Rounds:
· Timing: The first round of the match and the first round after teams switch sides
· Restrictions:
  · Only pistol-type weapons can be used
  · Weapon Mastery system is disabled
  · Ultimate system is disabled (no charging, cannot be used)
· Purpose: Opening rounds based on pure aiming skill that determine economic advantage
Normal Rounds:
· All rounds except pistol rounds
· Weapon Mastery and Ultimate systems are active

## Buff System and Stat Modifications
Buffs improve weapon base values by percentage or fixed values. Damage never changes.
Example: “Viper” Assault Rifle:
· Base (Level 1): Reload: 2.5s | ADS: 0.30s | Fire Rate: 600 RPM
· Level 2: ADS speed increases by 10% (0.30s → 0.27s)
· Level 3: Reload speed increases by 15% (2.5s → 2.12s)
· Level 4: Movement speed increases by 5% (250 → 262.5 units/s)
· Level 5: Fire Rate increases by 10% (600 RPM → 660 RPM)
Calculation Logic:
```
FinalValue = BaseValue × (1 ± BuffModifier)
```
· For times (Reload/ADS): Value decreases (1 – Modifier)
· For speeds (Movement/Fire Rate): Value increases (1 + Modifier)

Critical Scenarios and Rules
Dynamic Level Decrease:
   · If a player at Level 3 (2050 XP) receives a cold streak penalty (-60 XP), XP drops to 1990
   · Result: Player immediately drops to Level 2 and loses Level 3 buffs
Pistol Round Exception:
   · Rounds 1 and 13 (or round after side switch) are marked as “Pistol Round”
   · No XP calculation in these rounds, but kill counter continues
Minimum XP Protection:
   · XP value can never drop below 0
Ultimate XP Logic:
   · XP is added to the most expensive weapon in inventory
   · If primary weapon is not available, added to pistol

# Section 4: Network Architecture and Synchronization
## Communication Protocol
· Pure UDP used
· Built on Unreal Engine 5 networking framework
Bit Packing and Quantization:
```
CompressedValue = ((WorldPosition – MinMap) / (MaxMap – MinMap)) × 65535
```
Time Synchronization
```
RTT = t2 – t1 (Round Trip Time)
TimeOffset = ServerTime – (ClientTime + RTT/2)
```
## Client-side Prediction
· Player moves immediately on input without waiting for server confirmation
## Server Reconciliation
· Server processes client input
· Client compares with server result
· Position corrected if error exceeds threshold
## Entity Interpolation
· Other players are rendered at their past positions
· Linear interpolation used
## Lag Compensation
· Server stores last 1 second of player positions
· When player shoots, server rewinds time and checks hit
Weapon Mastery Data Replication
· Data sent only when changed
· Delta compression used (only difference sent)

## Wallbang Algorithm
1. First hit point detected
2. Exit point found (ray marching technique)
3. Thickness calculated
4. Damage reduction applied
Damage Falloff Formula:
```
FinalDamage = BaseDamage × (1 – (Thickness × Density) / PenetrationPower)
```
Critical Gameplay Rules
1. Corner Shots: Count as wallbang with minimum damage reduction
2. Consecutive Walls: Bullet can penetrate multiple walls (damage reduces each time)
3. Glass and Breakable Objects: Minimum resistance, break effect triggered

Capsule-Ray Mathematics
1. Shortest distance between bullet ray and capsule axis calculated
2. Collision checked (distance ≤ radius)

· A bullet can penetrate multiple capsules (e.g., hand then head)
· Priority Rule: Zone with higher damage multiplier is selected
  · Example: If both hand and head are hit, counted as headshot

# Section 11: 3D Spatial Audio Engine
## Distance and Attenuation
```
Gain = ReferenceDistance / (ReferenceDistance + (Distance × RolloffFactor))
```
· ReferenceDistance: 100% sound (e.g., 1 meter)
· RolloffFactor: How quickly sound fades
· Maximum Range: Not processed if gain < 0.01
3D Positioning
· Pan Calculation: Pan = SoundVector · RightVector
  · 1.0: Far right
  · -1.0: Far left
  · 0.0: Center
· Front/Back Differentiation: Rear sounds filtered with low-pass
Dynamic Occlusion
· Raycast between sound source and listener
· No Obstruction: Occlusion = 0.0
· Obstruction Present: Value based on material
  · Wood: 0.4
  · Concrete: 0.7
  · Metal: 0.9
Filter Effect:
```
CutoffFrequency = 22000Hz × (1.0 – Occlusion)
```
· No obstruction: 22kHz (full sound)
· Concrete wall: 6kHz (muffled sound)

Channel Priority Order:
1. Critical: Sphere, headshot, own damage sound
2. Voice Comms: Team chat
3. Ultimate Cues: “Fire in the hole!”

5. Footsteps
6. Ambience: Wind, birds, map hum
